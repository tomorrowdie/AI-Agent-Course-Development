# Output Format Templates (follow EXACTLY — do not add sections)

Two approved formats. Ask the user which one applies (or infer: institutional/government submission → Format 1 + Format 2 syllabus block). Do not invent your own structure.

## Format 1 — Course Execution Document (government/institution submission)

Modeled on an accepted government training submission. Sections, in order (language: match the user's audience — Traditional Chinese for HK government, or bilingual):

1. **課程名稱 / Course title** + delivery context line (venue, audience, constraints e.g. no internet)
2. **課程一句話定位** — ONE sentence stating what learners will master and why it matters for employment
3. **課程執行原則** — how the course follows the required framework (if a mandated structure like S1–S11 exists, state it will be executed without modification; adaptations only in delivery method)
4. **課程核心概念** — 1–2 short paragraphs on the teaching approach
5. **適合對象** — plain bullet list of learner types
6. **課程目標** — bullet list of concrete objectives (verbs: 理解/掌握/懂得/完成)
7. **課程大綱表** — table: Session | 課程主題 | 教學重點 (one row per session; teaching points as a comma-separated phrase list, NOT paragraphs)
8. **每堂課建議教學流程** — numbered 4–6 step flow (concept intro → case/demo → group work → sharing → tutor feedback → project accumulation)
9. **評核安排** — table: 評核項目 | 比重 | 內容, ending 總計 100%
10. **教學團隊可提供的支援** — short bullet list

Rules: concrete over abstract, short bullets over paragraphs, no jargon, no invented statistics or case names. If a constraint exists (no internet, no computers), state the offline alternative explicitly.

## Format 2 — Formal Syllabus block (sections V–VIII, university/accreditation style)

Use these exact section numbers and headers:

**V. Syllabus** — numbered modules; under each, plain bullet topics (5–8 bullets max per module). Topic phrases only, no sentences.

**VI. Teaching and Learning Activities** — one line, hours by type, e.g.: `Lectures: 33 hours; Computer Lab Practices: 7 hours.` Hours MUST sum to the course total.

**VII. Assessment** — table: Type of Assessment | Description | Weighting | ILO. Description includes duration, group size, word counts. Weightings sum to 100%. ILO column maps to learning outcome numbers.

**VIII. Required and Recommended Reading** — Harvard/APA convention. CRITICAL: never invent books, authors, or URLs. Only cite readings found in the provided knowledge or supplied by the user; otherwise write `[Readings to be confirmed by programme team]`.
